  Bug Description                                                               
                                                                                
  When using a provider configured with api: openai-responses, the gateway      
  sends store=false or omits store, which defaults to false. On subsequent      
  turns within the same session, the gateway includes previous_response_id      
  referencing the previous response. Since that response was never persisted    
  server-side, the API returns a 404.                                           
                                                                                
  HTTP 404: Item with id "rs_xxxxx" not found. Items are not persisted when     
  store is set to false. Try again with store set to true, or remove this item  
  from your input.
