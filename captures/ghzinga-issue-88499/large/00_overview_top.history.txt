  Bug Description                                                                                                                                               
                                                                                                                                                                
  When using a provider configured with api: openai-responses, the gateway sends store=false in responses API requests.                                         
  On subsequent turns within the same session, the gateway includes previous_response_id referencing the previous                                               
  response. Since that response was never persisted server-side, the API returns a 404.                                                                         
                                                                                                                                                                
  HTTP 404: Item with id rs_xxxxx not found. Items are not persisted when store is set to false. Try again with store                                           
  set to true, or remove this item from your input.                                                                                                             
                                                                                                                                                                
  Expected Behavior                                                                                                                                             
                                                                                                                                                                
  The provider should either avoid previous_response_id when store=false or persist the response when continuing a
