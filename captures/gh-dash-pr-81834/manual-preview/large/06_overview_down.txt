                                                                        │  ## User-visible / Behavior Changes
                                                                        │  • `senseaudio` appears in `openclaw infer tts providers`, "configured" when
                                                                        │  `SENSEAUDIO_API_KEY` is in env or `messages.tts.providers.senseaudio.apiKey`.
                                                                        │  • `openclaw infer tts voices --provider senseaudio` lists the upstream system
                                                                        │  voices.
                                                                        │  • `openclaw infer tts convert --model senseaudio/senseaudio-tts-1.5-260319 ...`
                                                                        │  returns MP3 (or opus for voice-note channel targets).
                                                                        │  • `autoSelectOrder: 45` — strictly behind ElevenLabs (20), Azure (30), MiniMax
                                                                        │  (40).
                                                                        │  Existing user defaults are not displaced.
                                                                        │  • Defaults: voice `female_0033_b`, model `senseaudio-tts-1.5-260319`, base URL
                                                                        │  `https://api.senseaudio.cn`, MP3 32 kHz / 128 kbps / stereo.
                                                                        │  • No new env vars / config keys / CLI flags. `SENSEAUDIO_API_KEY` was already
                                                                        │  declared by the ASR side.
